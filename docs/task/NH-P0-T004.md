## 7. Next.js Application Creation

Create the app using the current stable Next.js CLI compatible with the installed Node.js LTS version.

Preferred command from the repository root:

```bash
pnpm dlx create-next-app@latest apps/web \
  --ts \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --use-pnpm \
  --import-alias "@/*"
```

When prompted:

- Use App Router: Yes
- Use Turbopack for local development: Accept the stable default
- Use `src/` directory: Yes
- Use Tailwind CSS: Yes
- Use TypeScript: Yes

Important:

- Do not create a nested Git repository.
- Remove a nested `pnpm-lock.yaml` if generated.
- Do not use npm or yarn.
- Ensure the app participates in the root pnpm workspace.
- Do not overwrite an existing application without inspection.

Set the package name in `apps/web/package.json`:

```json
{
  "name": "@nexthire/web",
  "private": true
}
```
