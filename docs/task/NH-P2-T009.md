## 7. Required Cross-Feature Invariants

Verify and enforce:

1. Only active authorized users can use protected APIs.
2. Candidates cannot access management APIs.
3. Management users require correct authoring/publish permissions.
4. Candidate-owned records are scoped by authenticated candidate ID.
5. Draft, archived, retired, internal, and invite-only assessments do not leak into candidate catalog.
6. Only published available assessments can start new attempts.
7. One active attempt exists per candidate and assessment.
8. Attempt snapshots are immutable.
9. Source edits never alter existing attempts, results, analytics, or certificates.
10. Correct answers, accepted answers, and explanations do not appear before finalization.
11. Client cannot submit score, attempt count, candidate ID, deadline, or cooldown values.
12. Server time controls deadlines and cooldowns.
13. Answer writes after finalization fail.
14. Submission is idempotent and concurrency-safe.
15. Scoring uses immutable snapshot data.
16. Score totals and counts reconcile.
17. Result review uses persisted scoring and does not silently rescore.
18. Analytics uses finalized stored result data.
19. Leaderboard participation is opt-in.
20. Leaderboards expose only approved public identity.
21. Retake limits include the first attempt.
22. Attempt numbers are unique and immutable.
23. One certificate exists per eligible passed attempt.
24. Certificate PDF uses immutable snapshot data.
25. Certificate files are private.
26. Signed URLs are short-lived and never stored or logged.
27. Raw certificate verification codes are never stored or logged.
28. Public verification exposes minimal data.
29. Audit logs contain no answers, correctness keys, signed URLs, tokens, or private profile data.
30. All timestamps use UTC.
