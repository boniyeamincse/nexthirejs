## 1. Objective

Create the initial NextHire monorepo structure using **pnpm workspaces** and **Turborepo**.

This task will only prepare the repository foundation. It will not create the Next.js web app, NestJS API, Flutter app, PostgreSQL, Redis, or Docker services yet.

---

## 2. Why This Task Is Required

NextHire will contain multiple applications and shared packages:

- Next.js web frontend
- NestJS backend API
- Flutter mobile application
- Shared TypeScript types
- Shared validation schemas
- Shared constants
- Infrastructure and documentation

A monorepo will keep these parts organized and make future development, testing, CI/CD, and shared package management easier.

## 7. Required Root Configuration

### `package.json`

Requirements:

- Private workspace
- Package manager pinned to the installed pnpm major version
- Node.js engine set to an active LTS-compatible range
- Scripts:
  - `build`
  - `dev`
  - `lint`
  - `test`
  - `typecheck`
  - `clean`

The scripts should delegate to Turborepo, even though application packages are not created yet.

Example behavior:

```json
{
  "private": true,
  "scripts": {
    "build": "turbo run build",
    "dev": "turbo run dev",
    "lint": "turbo run lint",
    "test": "turbo run test",
    "typecheck": "turbo run typecheck",
    "clean": "turbo run clean && rm -rf node_modules"
  }
}
```

Add Turborepo as a root development dependency.

### `pnpm-workspace.yaml`

Include:

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

The Flutter application will live in `apps/mobile`, but pnpm must not attempt to manage Flutter dependencies. Do not add a Node `package.json` inside `apps/mobile` during this task.

### `turbo.json`

Create tasks for:

- build
- dev
- lint
- test
- typecheck
- clean

Requirements:

- `build` depends on dependency builds
- `dev` is persistent and not cached
- `clean` is not cached
- Outputs may remain empty until applications are initialized

### `.npmrc`

Include safe workspace defaults:

```ini
auto-install-peers=true
strict-peer-dependencies=false
save-workspace-protocol=rolling
```

### `.editorconfig`

Use:

- UTF-8
- LF line endings
- Final newline
- 2 spaces for JSON, YAML, JavaScript, and TypeScript
- 4 spaces for Dart
- Trim trailing whitespace except Markdown

### `.gitignore`

Cover at minimum:

- Node modules
- Turborepo cache
- Environment files
- Build output
- Coverage
- Logs
- IDE files
- OS files
- Flutter and Dart generated files
- Local Docker data
- Temporary files

Do not ignore:

- `.env.example`
- Documentation files
- Lockfile
