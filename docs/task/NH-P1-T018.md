## 5. Required Cross-Feature Invariants

Verify and enforce:

1. Only active accounts use protected candidate features.
2. Suspended, deleted, and deactivated accounts cannot login, refresh, or access protected APIs.
3. Every self-service query is scoped by authenticated user ID.
4. Client payloads never control self-service user IDs.
5. Candidate-role enforcement is consistent.
6. Public endpoints never expose email, date of birth, auth, session, token, audit, or hidden profile data.
7. Hidden profile sections are omitted by the backend.
8. Privacy settings do not change stored completion.
9. Completion points total exactly 100.
10. Reordering never changes completion.
11. Raw passwords, tokens, token hashes, cookies, signed URLs, and authorization headers never appear in logs.
12. Raw verification, reset, refresh, share, and download tokens are never stored.
13. Revoked sessions cannot refresh or access protected APIs.
14. Password reset revokes all sessions.
15. Password change revokes all other sessions and preserves the current session.
16. Deactivation revokes every session and disables public/link-only access.
17. Data export contains only the candidate's safe data and excludes secrets.
18. Queue jobs are bounded and idempotent.
19. Audit metadata contains no sensitive payload values.
20. Frontend does not persist tokens unsafely or calculate the authoritative completion score.
