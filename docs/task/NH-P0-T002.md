- Next.js web application: local process
- NestJS API application: local process
- Flutter application: emulator or physical device

Only supporting infrastructure will run in Docker:

- PostgreSQL 16
- Redis 7
- MinIO object storage
- Mailpit local email server

## 9. Docker Compose Requirements

Create:

```text
infrastructure/docker/compose.dev.yml
```

Use Docker Compose specification syntax without the deprecated top-level `version` field.

### 9.1 PostgreSQL Service

Service name:

```text
postgres
```

Requirements:

- Image: `postgres:16-alpine`
- Container name derived from the Compose project where possible; avoid unnecessary hard-coded container names
- Restart policy suitable for local development
- Port mapping from `${POSTGRES_PORT}` to `5432`
- Environment variables:
  - `POSTGRES_DB`
  - `POSTGRES_USER`
  - `POSTGRES_PASSWORD`
- Named volume for `/var/lib/postgresql/data`
- Health check using `pg_isready`
- Attach to the dedicated development network

### 9.2 Redis Service

Service name:

```text
redis
```

Requirements:

- Image: `redis:7-alpine`
- Port mapping from `${REDIS_PORT}` to `6379`
- Persistent named volume for Redis data
- Start Redis with append-only persistence enabled
- Health check using `redis-cli ping`
- Attach to the dedicated development network
- Do not configure production authentication in this local-only task

### 9.3 MinIO Service

Service name:

```text
minio
```

Requirements:

- Use the official MinIO image
- Start with:
  - API port `9000`
  - Console port `9001`
- Port mappings:
  - `9000:9000`
  - `9001:9001`
- Environment variables:
  - `MINIO_ROOT_USER`
  - `MINIO_ROOT_PASSWORD`
- Persistent named volume for MinIO data
- Attach to the dedicated development network
- Add a health check against the MinIO live-health endpoint when supported by the selected image
- Document that the image tag is for local development and must be pinned during production deployment planning

### 9.4 MinIO Bucket Initialization

Create a one-shot service named:

```text
minio-init
```

Requirements:

- Use the official MinIO client image
- Wait until MinIO is healthy
- Create the bucket from `${MINIO_DEFAULT_BUCKET}` if it does not already exist
- Do not make the bucket public
- Exit successfully when the bucket already exists
- Attach to the same Docker network

The initialization command must be idempotent.

### 9.5 Mailpit Service

Service name:

```text
mailpit
```

Requirements:

- Use the official Mailpit image
- SMTP port: `1025`
- Web interface port: `8025`
- Map both ports to localhost
- Attach to the dedicated development network
- No production email provider configuration

### 9.6 Network

Create one named network:

```text
nexthire-dev
```

All services must use it.

### 9.7 Volumes

Create named volumes for:

```text
postgres-data
redis-data
minio-data
```

Do not bind database data to arbitrary local directories.

### 14.6 Verify PostgreSQL

```bash
docker compose --env-file .env -f infrastructure/docker/compose.dev.yml exec -T postgres \
  pg_isready -U "$POSTGRES_USER" -d "$POSTGRES_DB"
```

Because shell variables may not be exported from `.env`, the coding agent may instead read them safely from the Compose container environment or run:

```bash
docker compose --env-file .env -f infrastructure/docker/compose.dev.yml exec -T postgres \
  sh -lc 'pg_isready -U "$POSTGRES_USER" -d "$POSTGRES_DB"'
```

### 14.7 Verify Redis

```bash
docker compose --env-file .env -f infrastructure/docker/compose.dev.yml exec -T redis \
  redis-cli ping
```

Expected result:

```text

```

### 14.8 Verify MinIO



### 14.9 Verify Mailpit

## 15. Required Health and Reliability Rules

- PostgreSQL-dependent application work must later wait for a healthy PostgreSQL service.
- MinIO bucket initialization must wait for healthy MinIO.
- `minio-init` must be safe to run repeatedly.
- Service restart must not delete data.
- `infra:down` must not remove volumes.
- Compose configuration must remain valid when started more than once.
- The task must not expose these development services to external interfaces intentionally.
- Do not use host networking.
- Do not use privileged containers.

Where supported, bind published ports to `127.0.0.1` so services are local-only, for example:

```yaml
ports:
  - "127.0.0.1:${POSTGRES_PORT}:5432"
```

Apply the same local-only binding approach to Redis, MinIO, and Mailpit.
