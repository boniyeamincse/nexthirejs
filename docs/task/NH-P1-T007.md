## 5. Business Rules

Implement these exact rules:

1. Only authenticated users with the `candidate` role can access this feature.
2. A candidate can access only their own preferences.
3. Each candidate has at most one preference record.
4. Supported countries are database-driven.
5. Only active supported countries may be selected.
6. Initial supported countries are Bangladesh, Pakistan, and India.
7. Country code uses ISO 3166-1 alpha-2 uppercase format.
8. Current city is required for this task.
9. Current city is free text and normalized; a city master is out of scope.
10. At least one preferred job role is required.
11. A maximum of five preferred job roles is allowed.
12. Preferred job roles are trimmed, normalized for duplicate detection, and stored without duplicate values.
13. At least one work mode is required.
14. Allowed work modes are `ONSITE`, `HYBRID`, and `REMOTE`.
15. At least one employment type is required.
16. Allowed employment types are `FULL_TIME`, `PART_TIME`, `CONTRACT`, `INTERNSHIP`, and `FREELANCE`.
17. Arrays must contain unique values.
18. The API must reject unsupported enum values.
19. Profile completion must be recalculated after a successful update.
20. The backend completion result is authoritative.
21. Email and authentication fields remain read-only and outside this payload.
22. User ID must never be accepted from the request body.
23. Audit metadata must not include full location or role-preference values.
24. API authorization is authoritative; frontend hiding is not security.
25. Suspended or deleted accounts cannot update preferences.
26. All timestamps use UTC.
27. Unexpected errors must not expose Prisma or stack-trace details.
28. The preference update must not modify another candidate's profile.
29. Preference storage must preserve user-entered display casing for job roles while using normalized comparison for duplicates.
30. Empty strings must not be stored in arrays.

## 7. Candidate Preference Design

Implement the logical equivalent of:

```prisma
enum WorkMode {
  ONSITE
  HYBRID
  REMOTE
}

enum EmploymentType {
  FULL_TIME
  PART_TIME
  CONTRACT
  INTERNSHIP
  FREELANCE
}

model CandidatePreference {
  id                       String           @id @default(uuid()) @db.Uuid
  userId                   String           @unique @db.Uuid
  countryId                String           @db.Uuid
  currentCity              String           @db.VarChar(100)
  preferredJobRoles        String[]
  preferredWorkModes       WorkMode[]
  preferredEmploymentTypes EmploymentType[]
  createdAt                DateTime         @default(now())
  updatedAt                DateTime         @updatedAt

  user    User    @relation(fields: [userId], references: [id], onDelete: Cascade)
  country Country @relation(fields: [countryId], references: [id], onDelete: Restrict)

  @@index([countryId])
}
```
