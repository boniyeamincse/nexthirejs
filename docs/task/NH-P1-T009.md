## 6. Business Rules

Implement these exact rules:

1. Only authenticated users with the `candidate` role can use this feature.
2. A candidate can access only their own experience records.
3. A candidate may have multiple experience records.
4. Maximum 30 experience records per candidate.
5. `companyName` is required.
6. `jobTitle` is required.
7. `employmentType` is required.
8. `startDate` is required.
9. `endDate` is optional only when `currentlyWorking` is true.
10. `endDate` is required when `currentlyWorking` is false.
11. `endDate` cannot be earlier than `startDate`.
12. Future start dates are not allowed.
13. `location` is optional.
14. `isRemote` is optional and defaults to false.
15. `responsibilities` is optional and length-limited.
16. `achievements` is optional and length-limited.
17. Duplicate records are not silently merged.
18. Ordering is candidate-controlled.
19. Reorder input must contain only owned record IDs.
20. Delete behavior must follow repository retention conventions; document hard delete or soft delete.
21. Profile completion is recalculated after create, update, delete, and reorder when ordering affects display only.
22. Audit metadata must not contain company name, job title, responsibilities, achievements, dates, or location.
23. API authorization is authoritative.
24. Suspended or deleted accounts cannot modify records.
25. All timestamps use UTC.
26. Cross-user access returns controlled not-found/forbidden behavior.
27. Request bodies must not accept arbitrary user IDs.
28. Empty optional strings normalize to `null`.
29. Git commit occurs only after all mandatory checks pass.
30. A blocked task must not create a commit.

model WorkExperienceRecord {
