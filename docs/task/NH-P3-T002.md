## 8. Expert Expertise

Suggested relation:

```prisma
enum ExpertExpertiseLevel {
  INTERMEDIATE
  ADVANCED
  EXPERT
}

model ExpertExpertise {
  id              String               @id @default(uuid()) @db.Uuid
  expertProfileId String               @db.Uuid
  expertiseAreaId String               @db.Uuid
  level           ExpertExpertiseLevel
  yearsExperience Int?
  isPrimary       Boolean              @default(false)
  createdAt       DateTime             @default(now())
  updatedAt       DateTime             @updatedAt

  @@unique([expertProfileId, expertiseAreaId])
  @@index([expertProfileId, isPrimary])
  @@index([expertiseAreaId, level])
}
```

Rules:

1. Minimum 1 expertise area before a service can be activated.
2. Maximum 20 expertise areas.
3. Maximum 5 primary expertise areas.
4. Years of experience is optional and must be 1–60 when provided.
5. Expertise area must be active.
6. Duplicate expertise is rejected.
7. Service must reference one of the Expert’s selected expertise areas.
8. Removing expertise used by an active service is blocked.
9. Removing expertise used only by inactive services requires either reassignment or service update.
10. Expertise level is self-declared but associated with the approved Expert identity; do not label it independently verified unless verification workflow supports that claim.

model ExpertWeeklyAvailability {

11. Store recurring windows as local wall-clock values, not converted fixed UTC timestamps.

## 15. Date-Specific Availability Overrides

Implement:

```text
UNAVAILABLE
CUSTOM_HOURS
```

Suggested:

```prisma
enum ExpertAvailabilityOverrideType {
  UNAVAILABLE
  CUSTOM_HOURS
}

model ExpertAvailabilityOverride {
  id                    String   @id @default(uuid()) @db.Uuid
  availabilityProfileId String   @db.Uuid
  localDate             DateTime @db.Date
  type                  ExpertAvailabilityOverrideType
  reason                String?  @db.VarChar(300)
  createdAt             DateTime @default(now())
  updatedAt             DateTime @updatedAt

  windows ExpertAvailabilityOverrideWindow[]

  @@unique([availabilityProfileId, localDate])
  @@index([availabilityProfileId, localDate])
}

model ExpertAvailabilityOverrideWindow {
  id         String @id @default(uuid()) @db.Uuid
  overrideId String @db.Uuid
  startLocalMinutes Int
  endLocalMinutes   Int

  @@index([overrideId, startLocalMinutes])
}
```

Rules:

1. One override per local date.
2. `UNAVAILABLE` has no windows.
3. `CUSTOM_HOURS` has at least one window.
4. Custom windows follow the same overlap/length rules.
5. Override replaces recurring weekly availability for that local date.
6. Past override creation may be rejected; allow editing only for today/future based on Expert timezone.
7. Recommended maximum future override range: 365 days.
8. Reason is optional and private.
9. Public/candidate availability views later must not expose private reason.
