# Assessment Leaderboard Rules

## Participation
- Default: disabled
- Enable: explicit candidate action via settings UI
- Display name: optional, 2-80 chars, trimmed; no uniqueness required
- Missing display name: uses safe generated alias (Candidate-XXXX)
- Disabling: immediate removal from all leaderboards

## Assessment Leaderboard
- Only opted-in active candidates
- Best finalized attempt per candidate per assessment
- Order: percentage (desc) → score earned (desc) → unanswered (asc) → duration (asc) → submission time (asc) → attempt ID

## Category Leaderboard
- Opted-in candidates with ≥1 completed assessment in category
- Unique assessment count (best attempt per assessment)
- Order: average percentage (desc) → unique assessment count (desc) → pass rate (desc) → best percentage (desc) → latest submission (asc) → candidate ID

## Privacy
- No email, phone, date of birth, location, or profile data exposed
- No candidate database IDs
- Audit logs exclude display alias text and private identity
